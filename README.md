## Overview

This repository is a comprehensive learning resource for growing developers who want to master project documentation using VS Code and GitHub. It demonstrates how to create professional-grade documentation that provides clear, rapid insights to both developers and their communities.

Whether you're documenting an open-source project, building internal wikis, or creating technical whitepapers, this repository provides the templates, best practices, and real-world examples you need.

## What You'll Learn

- 📝 **VS Code Best Practices**: Efficiently write and organize documentation with extensions, shortcuts, and workflows
- 🔄 **GitHub Workflows**: Master Git and GitHub for version control, collaboration, and publishing
- 📚 **Documentation Types**: Create READMEs, whitepapers, roadmaps, technical docs, and code documentation
- 🔒 **Public vs Private**: Understand security, privacy, and what to share publicly vs. keep private
- 🌍 **Community Building**: Write documentation that engages developers and builds thriving communities

## Quick Start

### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/Project-Documentation.git
cd Project-Documentation
```

### 2. Explore the Documentation

Start with the comprehensive guides in [`docs/`](docs/):

- **New to documentation?** → [Documentation Types Guide](docs/documentation-types.md)
- **Using VS Code?** → [VS Code Best Practices](docs/vscode-best-practices.md)
- **Learning Git/GitHub?** → [GitHub Workflows](docs/github-workflows.md)
- **Need security guidance?** → [Public vs Private](docs/public-vs-private.md)

### 3. Use the Templates

Browse ready-to-use templates in [`examples/`](examples/):

- [README Template](examples/README-template.md) - Comprehensive project README
- [Contributing Template](examples/CONTRIBUTING-template.md) - Contribution guidelines
- [Changelog Template](examples/CHANGELOG-template.md) - Version tracking
- [Security Template](examples/SECURITY-template.md) - Security policy

### 4. Apply to Your Projects

Copy templates to your projects and customize:

```bash
# Copy a template
cp examples/README-template.md ../your-project/README.md

# Open in VS Code
code ../your-project/README.md

# Customize placeholders like [Project Name], [Description], etc.
```

## Tools & Technologies

This repository teaches documentation using:

- **Editor**: [Visual Studio Code](https://code.visualstudio.com/) with Markdown extensions
- **Version Control**: [Git](https://git-scm.com/) and [GitHub](https://github.com/)
- **Format**: [Markdown](https://www.markdownguide.org/) (GitHub Flavored)
- **Diagrams**: [Mermaid](https://mermaid.js.org/), [Draw.io](https://www.drawio.com/)
- **Publishing**: [GitHub Pages](https://pages.github.com/), Static Site Generators

### Recommended VS Code Extensions

- **Markdown All in One** - Shortcuts and formatting
- **markdownlint** - Style consistency
- **Code Spell Checker** - Spell checking
- **Prettier** - Auto-formatting
- **GitLens** - Git history and blame

See [VS Code Best Practices](docs/vscode-best-practices.md) for complete setup guide.
